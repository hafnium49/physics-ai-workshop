# 3. 事前準備

このワークショップでは、**Visual Studio Code** と **Remote - SSH** 拡張機能を使って、GX10 ワークステーションに接続します。
標準の VS Code には SSH 接続機能は入っていないため、**Remote - SSH のインストールが必要です**。

---

## 3.1 Visual Studio Code をインストールする

### A. Windows 11 の場合

1. ブラウザで **`code.visualstudio.com`** を開く
2. **Windows 版インストーラー**をダウンロードする

   * 通常は **User Installer** で大丈夫です
3. ダウンロードしたファイルを実行する
4. 画面の案内に従ってインストールする
5. インストール後、VS Code を起動する

### B. Ubuntu in WSL2 の場合

WSL2 を使っている場合は、**WSL2 内に Linux GUI 版 VS Code をインストールしません**。
推奨手順は以下です。

1. **Windows 側に VS Code をインストール**する

   * 上の **A. Windows 11 の場合** と同じ手順でOKです
2. 必要に応じて VS Code に **WSL** 拡張機能も入れておく
3. WSL2 の Ubuntu ターミナルで、必要なら次のように起動できます

```bash
code .
```

この `code .` により、VS Code の画面は Windows 側で動きつつ、ワークスペースやターミナルは WSL2 側と連携します。

> **今回のワークショップでは、最終的には VS Code から GX10 に Remote-SSH で接続します。**
> そのため、VS Code は **Windows 側に入っていれば十分**です。

---

## 3.2 Remote - SSH 拡張機能を入れる

1. VS Code を開く
2. 左側の **拡張機能** アイコン（四角が4つ並んだマーク）をクリック
3. 検索欄に **`Remote - SSH`** と入力
4. **Microsoft** が公開している **Remote - SSH** を見つける
5. **Install** をクリックする

> WSL2 利用者は、必要に応じて **WSL** 拡張機能も入れておくと便利です。

   （または `Ctrl + Shift + P`）

3. 次に：

**+ Add New SSH Host...**

4. 次の形式で接続コマンドを入力する：

ssh -p 22222 engineerX@[IP_ADDRESS]

5. Enter を押す
6. SSH 設定ファイルの保存先を聞かれたら、最初に表示される候補を選ぶ
